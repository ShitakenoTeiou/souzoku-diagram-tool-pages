# prototype ver13

相続関係図ツールの ver13 仮版プロトタイプです。

## 目的

ver13 は完成版ではありません。

ver8までの確認内容に加えて、印刷プレビューの1ページ収まり、カードと線の比率、通常画面に近い印刷レイアウトを検証するためのプロトタイプです。

## 主な確認項目

- 人物情報の修正
- 親・子・配偶者の追加
- 関係性の修正
- 関係性の削除
- 直前1回のUndo
- 既存人物を使う配偶者追加
- 新規人物作成時の簡易重複チェック
- 実子追加時の不明親作成とcommonLawのSpouseRelation作成
- 実子追加時に選択人物の姓を初期入力
- 親追加で既存夫婦へ関係線のみ追加
- 養親側から新しい養子を追加する最小ルート
- commonLawの配偶者線を単線で表示
- 離婚の×を配偶者線の左側へ表示
- 人物削除の最小実装（被相続人から孤立する人物が出る削除は禁止）
- 既存ツリーに新しい兄弟姉妹を追加したときのツリー重なり回避
- 人物カード追加後も選択中カードを維持
- 人物情報の生年月日・死亡年月日入力
- 同じ子に実親線・養親線など複数の親子線が入る場合の接続位置分散
- 養子に実親を追加する場合、養子本人の姓を実親欄へ初期入力しない挙動
- 印刷プレビュー画面
- 通常版/簡易版、A4/A3、縦/横、1ページに収める印刷設定
- 印刷画面で横向き用紙を選んだ場合の横長プレビュー
- 1ページに収めるONのとき、印刷時も用紙内へSVGを固定して縮小表示
- 印刷画面の人物カード寸法と縦間隔を入力画面に近い比率へ調整
- 印刷画面から戻った場合の入力状態復元
- 複雑な家族構成の読み込み確認用サンプルJSON
- 親世代追加時の過剰な縦方向拡大抑制
- 1人目の子を配偶線・親セット線の中点から直線で接続する親子線

## ファイル構成

```text
prototype/ver13/
├─ index.html
├─ style.css
├─ app.js
├─ print.html
├─ print.css
├─ print.js
├─ README.md
├─ sample-complex-family-ver13.json
└─ sample-print-spouse-parents-ver13.json
```

庁内PCで試す場合は、同じフォルダに `index.html`、`style.css`、`app.js`、`print.html`、`print.css`、`print.js`、`sample-complex-family-ver13.json` を置いて `index.html` を Edge で開きます。

## この仮版で対象外にしていること

- 関係修正の完全版
- 養子追加時の実親登録の完全版
- 不明人物統合
- 複数回Undo
- 大人数案件での印刷縮尺調整
- 日付入力の和暦バリデーションと西暦変換表示
- 備考編集




## 追加サンプルJSON

sample-print-spouse-parents-ver13.json は、被相続人と妻の両親、妻の姉を登録した印刷配置確認用のサンプルです。夫婦ペアが親世代で分断されないこと、妻の姉が妻と同世代に配置されることを確認します。
